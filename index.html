Basics of the Web

  • Browser - A program that runs on your computer to display files found on the web
  • The Internet - The world's largest computer network
  • HTTP - stands for HyperText Transport Protocol, and is the main protocol of the web
  • Servers - Computers that host the files that make up the web

    • Files that make up the Web:

  • Plain Text
  • HTML
  • Images
  • Video
  • Music

    • Basics of HTML

    HTML - stands for Hyper Text Markup Language, and is made up of:

  • Text content - what you see
  • Markup - what it looks like
  • References to other documents - e.g. images and videos
  • Links to other pages

    • Basics of HTML Markup

    Tags are structured as follows:

    <name> content </name>

  • <name> - opening tag
  • contents - the text or files that you want the tag to affect
  • </name> - closing tag

    • This is also referred to as an Element.
    This type of tag will effect everything that follows until it is closed (</>)

    Example Tags

    <b>Bold Text Tag </b>
    <em>Italic Text Tag </em>
    <a>Anchor Tag </a>

    Anchor tags are often used to hyperlink to another webpage, looking like this:
    <a href = "URL"> Display Text </a>

    <img src = "URL" alt = "alternate text"> -is an Image Link Tag.
    This is called a Void Tag because it does not need a closer.

    Basics of Whitespace

    There are two types of whitespace:

  • Inline- Ends a line by wrapping text to a new line
  • Block-Adds an invisible box around the text

    • Basics of CSS:

    CSS - stands for Cascading Style Sheets, and is code written to control the "style" of HTML elements.

  • Inheritance - the mechanism by which properties are applied not only to a specified element, but also to it's descendants.

    • In general, descendant elements inherit text-related properties, but box-related properties aren''t inherited.
    Inheritance prevents certain properties from being declared over and over again in a style sheet, allowing software devs to write less CSS.

    Creating a Structured HTML Document

    DOM - stands for Document Object Model, and is the tree-like structure of an HTML document.

    On a website, everything you see is contained in a box. Boxes can be inside other boxes.
    When writing code, start with the bigger boxes and work down to the smaller ones to avoid confusion.

    Use the <div> </div> tag to define boxes.

    Code, Test, Refine

    1. Look for natural boxes ("boxify")
    2. Look for repeated styles and semantic elements
    3. Write your HTML
    4. Apply styles (from BIGGEST to smallest)
    5. Fix problems

    Structure of an HTML Document

    <!DOCTYPE HTML>
    <html>
          <head>
                <title> TITLE </title>
          </head>
          <body>
                <b> CONTENT </b>
          </body>
    </html>
